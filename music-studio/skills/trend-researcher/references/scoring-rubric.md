# Scoring rubric — 4 dimensions, 1–5 scale

Each candidate idea gets a score on four dimensions. Total isn't the average — the orchestrator looks at the shape (low-feasibility 5/5/5/5 idea is rejected).

## Click potential — would a stranger click?

| Score | Meaning |
|---|---|
| 1 | Topic is generic ("a synth"); no curiosity gap. |
| 2 | Niche but flat. ("A synth with 4 oscillators.") |
| 3 | A clear hook exists if framed well. |
| 4 | Strong evidence (Reddit thread, comment cluster) of unmet demand. |
| 5 | Multiple sources show real demand AND a clean curiosity-gap framing exists. |

Evidence trumps enthusiasm. Don't rate 5 because YOU think it's cool — rate 5 because you can show ≥ 2 sources of demand.

## Build feasibility — can it ship in 1–3 sessions?

| Score | Meaning |
|---|---|
| 1 | Multi-week project. Backend, persistence, ML model. |
| 2 | A week's work. New library to learn or unfamiliar audio architecture. |
| 3 | 2–3 sessions. Standard Tone.js + Web Audio territory. |
| 4 | 1–2 sessions. Reuses patterns from existing builds in this repo. |
| 5 | One session. A focused twist on something already built. |

**Floor: 3.** Anything below is a research project, not an episode.

## Musical payoff — does the demo sound good?

| Score | Meaning |
|---|---|
| 1 | The demo would sound bad or generic. (A "drum machine that plays exactly like a normal drum machine".) |
| 2 | Plausible demo, but a slight reach to make it sound musical. |
| 3 | Clear demo path with a satisfying loop. |
| 4 | The demo IS the title — the sound itself is the surprise. |
| 5 | The demo could carry the channel for a while; the build is a vehicle for repeated demos. |

## Shorts potential — does it cut into ≥ 2 viral-shaped clips?

| Score | Meaning |
|---|---|
| 1 | Nothing would short well. Talking-head explanation only. |
| 2 | One usable clip if you stretch. |
| 3 | 2 clear short candidates. |
| 4 | 3 clear short candidates including the demo drop. |
| 5 | The build has a "let me play with it for 30 seconds" property that produces shorts naturally. |

## How the orchestrator uses these

The orchestrator picks the candidate that **maximises the minimum score** across the four dimensions. A 5/5/5/2 candidate is worse than a 4/4/4/4 — the 2 will dominate the experience.

## Tie-breakers

When two candidates have the same minimum:

1. Prefer the one with higher musical-payoff (this is a music-tech channel).
2. Then prefer higher build-feasibility (cheaper to test).
3. Then prefer the one with the most-recent / most-upvoted source.
