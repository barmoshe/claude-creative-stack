# Thumbnail formulas — by episode type

Match the thumbnail formula to the episode type. Each formula exists because it's worked in the music-tech niche; replace these with sourced benchmarks from `channel-brain/packaging.md` once the channel has 5+ episodes' data.

## For build episodes (most common)

**Formula A — App + 2-word headline.**
- Background: solid color or subtle gradient (oklch values from the channel palette).
- Foreground left: app screenshot, cropped to the most recognisable element.
- Foreground right: 2–4 word headline, sans-serif, large.
- One pop element (a hand-drawn arrow, a circle, a "?").

**Formula B — Single weird visual.**
- The thing the title promises, made visible.
- No headline if the visual carries the meaning.
- Works only when the build's UI is striking on its own.

## For comparison episodes

**Formula C — Before/after split.**
- Vertical split: left half boring/expected, right half weird/cool.
- One line of text bridging the two ("but make it weird", "now do this").
- Arrow optional but usually helps.

## For "I tried X" episodes

**Formula D — Reaction face + result.**
- Creator on-camera, mid-reaction.
- App screenshot or weird-output overlaid right side.
- Headline 2–3 words.

## For deep-dive / explainer episodes (rare on this channel)

**Formula E — Annotated diagram.**
- The build's signal flow or core algorithm, drawn diagrammatically.
- 3–5 word headline at top.
- Use sparingly — explainer thumbnails underperform demo thumbnails on this channel.

## Universal rules across all formulas

- 1280 × 720 px.
- Headline readable at 350 px width in one second.
- Maximum 4 words of headline text.
- One core idea — if you're squinting to find the focus, the design has failed.
- App screenshot must be recognisable, not a code editor.
- Avoid overlaid logos (the YouTube UI already adds a watermark).

## Choosing between formulas

If the build's UI is the surprise → Formula A or B.
If the build's behavior is the surprise → Formula C.
If the creator's reaction is the surprise → Formula D.
If the algorithm is the surprise → Formula E (rare).
