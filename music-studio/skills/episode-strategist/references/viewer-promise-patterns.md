# Viewer-promise patterns

The viewer promise is one sentence: *"I built [X] that [surprising result]."* Pick the shape that best fits the build.

## Pattern 1 — Object → unexpected behavior

`I built a [familiar object] that [does something it shouldn't].`

- *I built a sequencer that writes impossible drum patterns.*
- *I built a synth that goes out of tune on purpose.*

Best when the build's twist is conceptual.

## Pattern 2 — Input → musical output

`I built a thing that turns [unlikely input] into [musical output].`

- *I built a thing that turns pencil drawings into MIDI.*
- *I built a thing that turns Wikipedia articles into ambient music.*

Best when the build's twist is the input modality.

## Pattern 3 — Constraint → surprising result

`I built [X] using only [tight constraint].`

- *I built a drum machine using only HTML buttons.*
- *I built a polyphonic synth in 100 lines of JavaScript.*

Best when the build's twist is the constraint itself.

## Pattern 4 — Question → answer-by-building

`What if [premise]? I built it.`

- *What if drums sequenced themselves on a Spirograph? I built it.*
- *What if you could mix a track with your voice? I built it.*

Best for exploratory premises.

## Pattern 5 — Comparison

`[Plugin/tool that costs $X], but [free / weird / browser-only].`

- *Like Output Arcade, but it lives in your browser tab.*

Use sparingly — the comparison must flatter, not nag.

## What none of these patterns say

The string "Claude" or "AI" does not appear in any pattern. The AI workflow is content for the body of the video, not the headline (Hard Rule 1).
