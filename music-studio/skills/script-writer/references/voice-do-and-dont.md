# Voice — do and don't

Replace these examples with phrases pulled from the creator's actual cuts after 5 episodes.

## Do

- "I built this thing that…"
- "Here's where it went sideways."
- "And this is the bit I'm proud of."
- "Watch what happens when…"
- "It shouldn't work, but…"
- Plain past tense for build narration: "I tried X, it broke, I fixed it by Y."

## Don't

- "In this video, we will…"
- "Today, I want to show you…"
- "Hi everyone, what's up."
- "Don't forget to like and subscribe!" — if you must, at the very end, briefly.
- "Game-changing", "revolutionary", "next-gen", "AI-powered" — adjectives that promise without specifying.
- Apologising in the script ("sorry for the delay"). Cut it.
- "We" when there's only one creator. "I" or "you".

## Word-level patterns

- Past tense for the build, present tense for the live demo, second person for handing the tool to the viewer:
  > "I built a sequencer (past). It plays an impossible polyrhythm (present). You can try it at the link below (second person)."

## Cold-open vocabulary

The cold open's first words should be one of:
- The demo audio (no words).
- A direct question.
- A weird claim.
- A reaction.

Never: "What's up", "Today", "In this video", "So I was thinking…"

## On bugs

- "It broke" beats "There was an issue".
- "I had no idea what I was doing" beats "I encountered some difficulties".
- The bug is content. Treat it like a punchline, not an apology.
