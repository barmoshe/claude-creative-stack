# Demo archetypes

The shape of the final episode demo. Pick one. They're not interchangeable — match the archetype to the build's nature.

## The Drop

```
intro (4–8 bars) → build (4–8 bars) → main loop (16 bars) → resolve (4 bars)
```

- Bars 1–8: hat or single element only.
- Bars 9–16: build adds layers, automation moves.
- Bars 17–32: full pattern, the wow moment lands here.
- Bars 33–36: filter close, kick out, tail.

**Use when:** the build has energy and the viewer expects payoff. Default for techno, IDM, electro builds.

## The Loop Showcase

```
loop A (4 bars) → smash cut → loop B (4 bars) → smash cut → loop C (4 bars) → ...
```

- Each loop demonstrates one feature.
- Hard cuts between loops, no transition.
- 4 bars is enough to land; longer is filler.

**Use when:** the build has multiple distinct features. Avoids the "demo-ing a list" problem by keeping each beat short.

## The Solo

```
one continuous take (32–64 bars), the build playing itself
```

- No edits.
- The build's autonomous behavior IS the demo.
- The voiceover is silent or sparse.

**Use when:** the build's magic is in what happens when you let it run. Generative, procedural, autonomous instruments.

## The Before/After

```
boring 4-bar baseline → 16–32 bar transformation
```

- Open with the obvious, expected version of what the build does.
- Smash cut to the build's actual output.
- The contrast IS the demo.

**Use when:** the value of the build is in how it differs from the default. "Like a normal sequencer, but…" builds.

## How to choose

| If the build's twist is… | Use |
|---|---|
| Energy / drop / surprise hit | The Drop |
| Many small features | The Loop Showcase |
| Autonomous behavior | The Solo |
| Contrast with the default | The Before/After |

## Forbidden

- A demo that ends on a fadeout. End on a hit, a tail, or a deliberate decay.
- A demo that fades up from silence over 2+ bars. The viewer has 2 seconds, use them.
- A demo with talking over the wow moment.
