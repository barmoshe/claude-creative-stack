Average INP target for hero interactions on a mid-tier laptop. This page sticks to transform, opacity, and a @property-typed gradient angle.

composite-only props. No height/top/margin in any animation.
scroll-driven. Sections fade in via animation-timeline: view(); gracefully degrades.
reduced motion. The hero hue spin pauses; section reveals collapse to instant.

Why this exists

Most "animated landings" still trigger layout on every frame and sneak localStorage into the artifact sandbox. This file is the canonical "you can do a real hero with three properties and one media query."

Try it

  1. Scroll. Sections rise in via the view-progress timeline.
  2. Open DevTools → Rendering → "Emulate CSS media feature prefers-reduced-motion". The hue spin freezes; fades collapse.
  3. Open Performance → Web Vitals lane. INP should sit comfortably under 200 ms.
